# GoBlocks — AI Agent Entry Point

GoBlocks is a premium WordPress Gutenberg block plugin built from scratch with TypeScript,
Zustand, @tanstack/react-query, PSR-4 PHP, and @wordpress/scripts. The goal is a
production-ready plugin that surpasses GenerateBlocks, Kadence Blocks, Spectra, Stackable,
and Greenshift in architecture quality, performance, and developer experience, and is
submitted to WordPress.org.

## Mandatory Agent Rules

### 1. Always start by reading PROGRESS.md
```
.claude/PROGRESS.md is the source of truth for what has been built.
Read it at the start of every session.
Update it (mark done / in-progress) before ending every session.
```

### 2. Always follow CODING-STANDARDS.md
Every file you create or edit must comply with `.claude/CODING-STANDARDS.md`:
- PHP: WPCS, `goblocks_` prefix, `GoBlocks\` namespace, `defined('ABSPATH') || exit;`
- TS/TSX: strict TypeScript, no `any`, no inline styles
- CSS: `gb-` class prefix, `--gb-` variable prefix, no `!important`
- i18n: all strings wrapped with text domain `'goblocks'`
- Escaping: always escape at output, never at input storage

### 3. No block production code without checking ATTRIBUTE-SCHEMA.md + ARCHITECTURE.md
Block `edit.tsx`, `save.tsx`, PHP render callbacks, and Inspector controls must
follow the nested `styles` object schema. Never create flat attribute triplets
(e.g., `widthTablet`) — always use `{ base, sm, md, ... }` pattern.

### 4. CSS rules
- Never write inline styles on block elements
- Never use `!important`
- Always use CSS custom properties for themeable values
- Block CSS is generated by the TypeScript CssEngine, stored in `attributes.generatedCss`
- PHP only collects + caches `generatedCss` — it never re-generates from `styles`

### 5. Block pattern: `save.tsx` always returns null
All GoBlocks blocks are dynamic (PHP render callback). `save.tsx` must always be:
```tsx
export function save(): null { return null; }
```
This eliminates the need for `deprecated.js` migrations.

### 6. Security
- Sanitize at input; escape at output
- All REST write endpoints check `current_user_can()`
- Never pass user input to dynamic `include()`/`require()`
- Use `$wpdb->prepare()` for all SQL
- Use `WP_Filesystem` for all file writes (CSS cache)

## Quick-Reference: Key File Paths

```
Main plugin:     goblocks.php
Uninstall:       uninstall.php
PHP classes:     includes/{Module}/{ClassName}.php
TS source:       src/{area}/{File}.ts(x)
Build output:    build/
Design tokens:   assets/css/tokens.css
Block pattern:   src/blocks/{name}/ + includes/Blocks/{Name}.php
Tests (PHP):     tests/php/
Tests (E2E):     tests/e2e/
```

## Quick-Reference: Key Commands

```bash
npm run build          # production build
npm run start          # watch mode
npm run lint:js        # ESLint
npm run lint:types     # TypeScript check (tsc --noEmit)
npm run test:unit      # Jest
npm run test:e2e       # Playwright
composer run phpcs     # WPCS lint
composer run phpstan   # static analysis (level 6)
composer run phpunit   # PHPUnit
npm run i18n:pot       # generate .pot file
```
